## Security Architect Observations

- **AKS hardening wave:** Multiple privilege-reduction and admission-control changes (kube-proxy capabilities, WireServer IP VAP, ClusterRoleBinding blocks, and Artifact Streaming/Pod Sandboxing validation) materially shrink the attack surface. Architects should validate existing pod security policies and RBAC models against these new defaults.
- **Ingress NGINX retirement:** The upstream Ingress NGINX project's end-of-maintenance creates a hard migration deadline. Architecture standards should adopt Gateway API as the default ingress pattern for all new AKS workloads and initiate retirement planning for NGINX-based implementations.
- **Identity modernization:** Entra ID’s accelerated push toward Cloud Sync, passkeys, and system-preferred authentication reduces long-term phishing and credential-theft risk, but requires hybrid identity roadmaps that account for feature parity gaps and phased rollout complexity.
- **AI agent governance:** The convergence of Microsoft Entra Agent ID, Foundry control plane, and Purview DSPM for AI indicates that non-human identity lifecycle and data exposure management are becoming first-class architecture concerns. Security architectures should define agent identity standards and data boundary controls now.
- **Data security expansion:** Purview’s GA data quality thresholds, advanced resource sets, and sensitivity label automation strengthen data governance foundations, while preview DLP capabilities (external domain blocking, unsaved file JIT) require careful scoping to prevent business workflow disruption.

## Security Operations Observations

- **AKS CVE-2026-31431:** This local privilege escalation affects non-root pods and requires urgent node image upgrades or DaemonSet mitigation. Operations teams should prioritize upgrade windows for existing node pools and verify image versions across all clusters.
- **Defender for Identity alert expansion:** Eight new Entra ID-focused alerts increase SOC coverage for identity-based attacks, including session cookie replay and Conditional Access bypass. Initial tuning periods should be planned to manage potential false positives.
- **Defender for Endpoint alert hygiene:** Built-in alert tuning rules are now GA and can suppress common benign noise without impacting AIR. SOC managers should enable and review these rules to improve queue efficiency.
- **Container runtime protection:** Defender Container Sensor v0.10.5 brings GA runtime antimalware detection and Bottlerocket support. Operations should stage sensor upgrades across clusters to close runtime detection gaps, especially on self-managed and baremetal deployments.
- **DLP operations impact:** New browser and network DLP email notifications, JIT unsaved-file protection, and mobile Outlook policy tips expand endpoint coverage. Test policies in monitoring mode first to gauge user impact and reduce helpdesk load.
- **Fabric monitoring gap:** Fabric Eventstream workspace monitoring requires republishing existing Eventstreams to emit telemetry. Operations teams should schedule republish cycles to avoid blind spots in real-time pipeline observability.
