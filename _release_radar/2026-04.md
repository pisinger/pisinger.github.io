## 🏛️ -> Security Architect Observations

- **Container security remains critical**: AKS faces multiple high-severity kernel and RBAC vulnerabilities (CVE-2026-33186, CVE-2026-31431, CVE-2025-1767) requiring rapid node image and cluster version upgrades. Architecture teams should enforce policy-as-code (Azure Policy) to block risky volume types and accelerate move to v1.33+ where built-in mitigations exist.
- **Government cloud parity advancing**: Defender for Containers now matches commercial feature coverage in Azure Government, enabling consistent multi-cloud/hybrid security posture for DoD and civilian agencies. Architects should review feature matrices for sovereign cloud deployments.
- **Identity infrastructure modernization**: Microsoft is actively pushing migration from Entra Connect Sync to Cloud Sync and expanding passkey/FIDO2 policy capacity. Architects should evaluate hybrid identity roadmaps, Source of Authority shifts, and phased workforce-to-External ID federation strategies.
- **AI agent governance emerging**: Microsoft Entra Agent ID (GA), Foundry Control Plane, and Defender XDR AIAgentsInfo table signal a maturing trust boundary for AI agents. Security architecture should incorporate OAuth 2.0, MCP, and A2A protocols into agent identity and authorization frameworks.
- **Ingress and service mesh transitions**: Upstream Ingress NGINX retirement and Istio revision deprecation require proactive traffic architecture planning. Gateway API adoption and Istio revision management should be included in cluster lifecycle standards.
- **OpenTelemetry standardization accelerating**: Azure Monitor and ACA both expanding OTLP/OTel support reduces vendor lock-in but requires unified observability architecture decisions for backends, collectors, and tenant isolation.

## 🛠️ -> Security Operations Observations

- **AKS node image hygiene is paramount**: Multiple Linux and Windows CVEs with confirmed exploits (non-root pod to root escalation) demand immediate node pool rotation and maintenance window coordination. SOC should validate that new node creation uses patched VHDs and track unpatched node inventories.
- **New identity attack surface alerts**: Defender for Identity added eight new Entra ID-focused alerts covering session cookie theft, Conditional Access bypass, and credential abuse. SOC should review detection coverage, tune false positive baselines, and ensure incident response playbooks cover identity compromise scenarios.
- **Alert tuning rules reduce noise**: Defender XDR built-in alert tuning rules are now GA for common benign activity. SOC teams should evaluate suppression policies carefully to avoid blind spots while reducing noise in Endpoint and Office 365 alerts.
- **Automated response status visibility**: Preview features in Defender XDR and Endpoint now show automatic attack disruption and predictive shielding action status in incident Activities tabs. SOC analysts should incorporate these statuses into triage workflows and verify containment outcomes.
- **AI agent monitoring expanding**: Defender XDR `AIAgentsInfo` table now covers Foundry, third-party, and custom agents. SOC should build hunting queries and detection rules for non-Copilot Studio agent activity, including anomalous authentication and resource access patterns.
- **DNS Detection for Kubernetes operationalized**: DNS Detection is now GA across AKS, EKS, and GKE. SOC should validate that Helm-deployed Defender sensors are capturing DNS telemetry and tune DNS-based detections for tunneling and C2 communication in containerized workloads.
