## 🏛️ Security Architect Observations

- **AKS node security model change**: The CVE-2026-31431 privilege escalation affects non-root pods with no special capabilities, indicating a kernel-level trust boundary failure. Architectures should enforce node image upgrade pipelines and avoid long-running static nodes. The shift from in-place patching to image-based replacement underscores the need for immutable infrastructure patterns.
- **Ingress NGINX to Gateway API migration**: The upstream NGINX retirement and AKS Gateway API GA represents a significant traffic management architecture shift. Security architects should review ingress policies, WAF configurations, and certificate management during migration, as Gateway API semantics differ from Ingress resources.
- **Defender portal unification**: The consolidation of Defender for Cloud into the Defender portal reduces tool sprawl but requires architectural planning around RBAC, data residency, and cross-cloud visibility. The risk-based Cloud secure score changes may alter how posture is communicated to executives.
- **Entra ID device and passkey resilience**: Soft-delete for device objects and expanded passkey policy storage improve identity resilience at scale. Architects should integrate device lifecycle management with backup and recovery strategies, and plan passkey profile segmentation for complex targeting scenarios.
- **Cross-tenant group synchronization**: While simplifying multi-tenant IAM, cross-tenant group synchronization introduces new trust boundaries. Architects must evaluate cross-tenant access policies and ensure synchronized groups do not bypass conditional access or privilege boundaries.
- **Defender for Identity sensor v3 expansion**: Full identity role support in v3 enables broader DC coverage, but the Windows Server 2025 migration blocker indicates version compatibility risks in heterogeneous environments. Sensor deployment planning should account for OS-specific support matrices.

## 🛠️ Security Operations Observations

- **Defender for Identity alert volume increase**: The nine new Entra ID-focused alerts (guest promotion, Global Admin creation, credential abuse, randomized user agents, stolen session cookies, CA bypass, and MFA tampering) will increase SOC triage volume. SOAR playbooks should be updated with automated enrichment and response workflows for these high-fidelity identity signals.
- **Sentinel UEBA data source expansion**: OktaV2 and GCP Audit Logs anomaly detections require verified connector configuration. Operations teams should validate data ingestion pipelines and tune baselines before relying on new detections for alerting.
- **Defender XDR automatic isolation operational impact**: Automatic device isolation, while reducing lateral movement, carries risk of operational disruption. SOC procedures should define rapid release protocols, whitelist critical business systems, and align isolation triggers with incident severity thresholds.
- **Defender Endpoint custom data collection scaling**: The increase to 75,000 events per device per day enables deeper telemetry but increases data volume and potential noise. Operations teams should audit existing rules for efficiency and monitor Log Analytics/Azure Monitor ingestion costs.
- **AKS runtime antimalware and privilege reduction**: The Container Sensor v0.10.5 GA runtime antimalware and kube-proxy reduced privileges improve container runtime security. Operations should validate DaemonSet rollout status and monitor for compatibility issues with existing pod security policies.
- **Purview DLP and DSPM new capabilities**: New external domain blocking and AI app monitoring for Claude expand data loss prevention scope. Operations should review policy coverage gaps, particularly for multimedia (MP4) content and emerging AI tool usage, and ensure scanner reporting meets compliance audit requirements.
