## Security Architect Observations

- **Unification of cloud security operations**: The integration of Defender for Cloud into the Defender portal with risk-based secure score and individual recommendations shifts posture management to a single pane of glass. Security architects should plan unified RBAC, data residency, and portal transition strategies.
- **Kubernetes add-on RBAC review**: The AKS `nodes/proxy` RCE path (AKS-2026-0004) demonstrates that Kubernetes add-on permissions can create critical attack surfaces even without assigned CVEs. Architecture teams should audit all add-on RBAC and prioritize Kubelet authorization model (KEP-2862) adoption on v1.33+.
- **DevSecOps bridge**: The native Defender for Cloud and GitHub Advanced Security integration closes the loop between runtime risk and code-level remediation. Architects should evaluate GHAS licensing, connector configuration, and Copilot Autofix governance models for CI/CD pipelines.
- **Cross-tenant trust boundaries**: Cross-tenant group synchronization in Entra ID simplifies multi-tenant collaboration but introduces new lateral movement paths. Review group lifecycle governance, attribute mappings, and cross-tenant access policies before broad enablement.
- **Phishing-resistant authentication momentum**: System-preferred first-factor authentication and expanded passkey policy storage in Entra ID remove barriers to passwordless adoption. Architects should design phased passkey rollout aligned to privileged access tiers.
- **Container security multicloud expansion**: Malware detection preview for EKS and GKE nodes, combined with Docker Hardened scanning, extends Microsoft's container security coverage beyond Azure. Multicloud architectures should validate sensor deployment models and image registry coverage.

## Security Operations Observations

- **Defender for Identity alert tuning**: Eight new Entra ID-focused alerts (guest promotion, global admin creation, stolen session cookies, MFA bypass) require SOC playbook updates, detection engineering validation, and stakeholder notification workflows.
- **Automatic containment workflow changes**: Automatic device isolation in Defender XDR/Endpoint alters incident response playbooks. SOC teams must define release procedures, communicate isolation scope to IT, and train analysts on the time-limited nature of the action.
- **Sentinel SOAR enhancements**: AI playbook generation and UEBA Okta V2/GCP anomaly expansions improve automation and cloud coverage. Validate OktaV2_CL connector health and review GCP anomaly baselines before relying on new detections.
- **Targeted malware scanning for Azure Files**: On-demand malware scanning for Azure Files enables ad-hoc and automated scanning. SOC teams should incorporate scans into insider threat, data exfiltration, and compliance response procedures.
- **Cloud security executive reporting**: New CNAPP Executive Summary and Cloud Posture reports in the Defender portal provide leadership-ready metrics. SOC leadership should align operational KPIs and remediation SLAs with these dashboards.
- **DLP policy scope review**: Updated Edge browser profile scope for unmanaged cloud app DLP policies means controls apply across personal and InPrivate profiles on managed devices. Review current DLP policies to avoid over-blocking or unexpected user impact.
