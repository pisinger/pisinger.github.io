## ⛵ AKS

| Indicator | Feature | Type | Description |
|---|---|---|---|
| 🔴 | CVE-2026-31431 kernel privilege escalation | Security | Linux kernel `algif_aead` vulnerability allows any pod to escalate to root on the underlying node. Mitigation is globally deployed in node image versions `202604.13.0` and `202604.24.0`; existing nodes must be upgraded or the mitigation DaemonSet applied immediately. |
| ⚫ | Ingress NGINX retirement | Deprecation | The Kubernetes SIG Network announced the upcoming retirement of the Ingress NGINX project with maintenance ending March 2026. AKS application routing add-on production workloads remain supported through November 2026. Migrate to the Gateway API implementation. |
| ⚫ | AKS Kubernetes LTS 1.29 deprecated | Deprecation | AKS Long Term Support version `1.29` is deprecated. Clusters must be upgraded to a supported version per the AKS Support Calendar. |
| 🔵 | AKS Kubernetes 1.32 LTS-only | Update | AKS version `1.32` is now available only through Long Term Support. Standard support clusters should upgrade or purchase an LTS plan. |
| 🟢 | Gateway API-based ingress GA | GA | Gateway API-based ingress for the application routing add-on is now generally available, replacing the soon-to-retire Managed NGINX ingress for new AKS Automatic clusters starting with Kubernetes 1.36. |
| 🟢 | AKS Automatic to Standard migration | GA | AKS Automatic clusters with managed system node pools can now migrate to AKS Standard clusters in additional regions after adding a system node pool. |
| 🟢 | `minReadySeconds` in Application Routing Gateway | GA | Users can now configure `spec.minReadySeconds` in the Application Routing Gateway Parameters ConfigMap to reduce disruption during rolling upgrades for apps needing extra initialization time. |
| 🟡 | NAT Gateway V2 outbound type | Preview | Preview support for AKS-managed NAT Gateway V2 outbound type is now available in supported public Azure regions. |
| 🟡 | Custom kubelet configuration | Preview | Preview of customization for default `kube-reserved` and hard eviction kubelet configuration via the custom node preview API starting with `2026-03-02-preview`. |
| 🟡 | Capacity Based Surge | Preview | Customers can now set both `MaxUnavailable` and `MaxSurge` values to surge during node pool upgrades based on available capacity. |
| 🟡 | AKS List Available VM SKUs API | Preview | New API to view VM SKUs supported on AKS and available in the subscription for cluster and node pool creation. |
| 🟡 | AKS-managed GPU metrics | Preview | GPU metrics are now supported by default in Azure Managed Prometheus and Grafana in Azure Monitor. |
| 🔴 | Azure Policy add-on CVE fixes | Security | Azure Policy add-on updated to `1.16.1` and Gatekeeper to `3.20.1-8` with CVE fixes. |
| 🔴 | Cost-analysis agent CVE fixes | Security | Cost-analysis agent and scraper images updated from `0.0.25` to `0.0.26` with CVE fixes. |
| 🟢 | Istio CRD installer registry fix | Fix | Fixed an issue where the Istio CRD installer could pull `busybox` from an unintended registry in AGC environments and removed non-Job Helm hooks to avoid race conditions. |
| 🟢 | Artifact Streaming + Pod Sandboxing validation | Fix | Added validation to prevent enabling Artifact Streaming with Pod Sandboxing, which is not supported. |
| 🟢 | AKS Automatic ClusterRoleBinding protection | Fix | Blocks `ClusterRoleBinding` create or update requests when the `roleRef` targets configured privileged `ClusterRole`s, reducing privilege escalation through service account impersonation. |
| 🔵 | kube-proxy reduced privileges | Update | AKS is rolling out reduced privileges for kube-proxy on Kubernetes `1.30` and later, using `NET_ADMIN` and `SYS_RESOURCE` capabilities instead of `privileged: true`. |
| 🔵 | HTTP Proxy CA certificate limit | Update | When using HTTP Proxy, you cannot add more than 20 Trusted CA certificates. |
| 🔵 | Mesh Membership Gateway API requirement | Update | Mesh Membership now requires the Managed Gateway API add-on to be enabled before a cluster can join an Azure Kubernetes Application Network. |
| 🔵 | Fleet-managed resource rollout | Update | Fleet-managed resources are now deployed through managed namespace `ClusterResourcePlacement` selection for separate rollout from customer workloads. |
| 🔵 | Component updates | Update | Multiple component updates: Istio revisions, Azure Monitor Container Insights `3.3.0`, Karpenter Azure provider `v1.10.2`, NGINX `1.13.9`, Azure Disk/File/Blob CSI drivers, cloud-provider-azure, Cilium `v1.17.10`, and new Windows/Azure Linux/Ubuntu node images. |

## 🏛️ Security Architect Observations

- **AKS CVE-2026-31431 is a critical container escape affecting non-root pods.** The ability for any pod to escalate to root on the node undermines container isolation boundaries. Organizations must treat node image lifecycle management as a first-class security control, not just a patching exercise, and ensure upgrade orchestration can be executed rapidly across all node pools.
- **Defender for Cloud integration into the Defender portal accelerates unified SecOps consolidation.** Microsoft's strategy to converge CSPM, threat protection, and XDR into a single pane reduces tool sprawl but requires architects to plan RBAC, data residency, and workflow migration carefully, especially for multicloud environments spanning AWS and GCP.
- **GitHub Advanced Security bidirectional integration bridges development and runtime security.** By connecting Defender for Cloud runtime signals (internet exposure, sensitive data access, lateral movement potential) with GHAS code-level alerts, organizations can finally prioritize developer backlog based on actual production risk rather than theoretical severity.
- **AKS Ingress NGINX retirement forces an ingress architecture migration.** The shift to Gateway API is not just a configuration change; it may require revisiting traffic management patterns, TLS termination, canary deployments, and third-party ingress controller dependencies across the Kubernetes estate.
- **Purview and Foundry enhancements signal a growing focus on AI data governance.** As organizations deploy AI agents and Copilots, the combination of DSPM, sensitivity labels for MP4/meeting artifacts, and Foundry Control Plane governance provides a defense-in-depth layer for data exposure in AI workloads.
- **Defender for Identity's new Entra ID alerts close hybrid identity detection gaps.** The addition of alerts for stolen session cookies, MFA bypass, and OAuth risks strengthens the bridge between on-premises AD and cloud identity threat detection, critical for organizations with hybrid identity architectures.

## 🛠️ Security Operations Observations

- **AKS CVE-2026-31431 requires immediate operational response.** SOC and platform teams must identify all node pools running affected images and schedule upgrades or DaemonSet mitigations without delay; this vulnerability is exploitable by non-privileged pods and bypasses standard container security assumptions.
- **Defender for Identity added eight new Entra ID-focused alerts.** SOC analysts should review the new alert types—including guest promotion to member, stolen session cookie replay, and suspicious third-party MFA additions—and update triage runbooks, escalation matrices, and false-positive tuning.
- **Sentinel AI playbook generation and UEBA enhancements expand SOAR and anomaly coverage.** The GA of AI-generated Python playbooks can reduce playbook development time, while Okta V2 and new GCP Audit Logs anomalies extend identity and cloud behavior monitoring; teams should validate integration and tune baselines.
- **Automatic device isolation in Defender XDR/Endpoint changes incident response workflows.** SOC operators must understand the conditions triggering auto-isolation, the network scope of isolation (blocks most traffic but preserves security service connectivity), and the release procedure to avoid disrupting business-critical systems during false-positive incidents.
- **Defender for Cloud daily score calculation now uses end-of-day snapshots.** Score trend analysis and SLA reporting may shift slightly due to the recalculation of historical values; SOC and GRC teams should communicate this change to stakeholders who track secure score KPIs.
- **Custom data collection GA in Defender for Endpoint raises the event limit to 75,000 per device per day.** While this enables richer telemetry for investigations and hunting, operations teams should monitor storage costs, ingestion rates, and endpoint performance impact when deploying high-volume custom rules.
