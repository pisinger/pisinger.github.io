## 🏛️ Security Architect Observations

- **AKS requires immediate multi-pronged hardening**: Multiple active CVEs span kernel modules, node images, ingress controllers, and Windows kernels. Architects should confirm automated node image rotation, enforce Azure Policy for deprecated volume types, and prioritize upgrades to Kubernetes 1.33+ to close the nodes/proxy RBAC gap.
- **Cross-tenant and multi-directory identity flows are expanding**: Entra ID cross-tenant group synchronization and External ID High Scale Compatibility mode introduce new trust boundaries. Architects must review attribute mappings, access policies, and group lifecycle governance to prevent lateral access creep across tenants.
- **MCP and AI agent endpoints form a new trust boundary**: With ACA and Fabric both hosting Model Context Protocol servers, architects should enforce ingress controls, managed identities, least-privilege secrets management, and OTel-based observability for these endpoints before broad adoption.
- **Cloud-native and code-to-runtime correlation strengthens DevSecOps**: Defender for Cloud’s GitHub Advanced Security integration creates bidirectional feedback loops between code vulnerability management and runtime exposure. Architects should inventory GHAS licensing, connector health, and data residency for this integration.
- **Data governance for AI agents requires forward planning**: Purview’s Agent 365 protections and sensitivity labels for groups/MP4s show Microsoft is extending governance to agentic content. Architects should align Fabric, Foundry, and Purview policies to ensure consistent classification and DLP coverage across human and AI-generated artifacts.
- **Network observability and service mesh security remain critical paths**: AKS updates to Cilium (CVE remediation), Istio gateway customization, and Static Egress Gateway private IPs reinforce the need for a well-defined pod-network segmentation strategy that includes eBPF observability and explicit egress controls.

---

## 🛠️ Security Operations Observations

- **AKS SOC playbook needs urgent refresh for nodes/proxy and kernel escalation**: The discovery that non-root pods can escalate to root via algif_aead auto-loading, combined with the nodes/proxy RBAC exposure, means SOC teams must validate detection logic for pod exec, node-level network connections to port 10250, and anomalous module loading events.
- **Defender for Identity alert expansion improves hybrid coverage**: New Entra ID alerts for guest promotion, session cookie theft, and suspicious MFA additions require immediate tuning in the Defender portal to avoid noise while capturing real credential abuse and token replay attempts.
- **Automatic attack disruption isolation introduces containment risk**: Both Defender XDR and Defender for Endpoint can now auto-isolate devices. SOC teams must define and test release procedures, create incident-specific communication plans, and validate that isolation does not interrupt critical Tier-0 services or management plane connectivity.
- **Sentinel AI playbook generation changes SOAR trust model**: Auto-generated Python automation requires rigorous code review, sandbox testing, and least-privilege service principals before production deployment to prevent supply-chain-like risks from AI-generated workflows.
- **Cloud security reporting consolidation demands retraining**: As Defender for Cloud migrates into the Defender portal with risk-based scoring and individual recommendation granularity, SOC analysts need updated training on navigation, new report types, and changed prioritization logic.
- **ACA Dynamic Sessions and self-hosted AI models create visibility gaps**: SOC teams must ensure that logs from sandboxed AI execution, serverless GPU jobs, and MCP server traffic are captured into Azure Monitor or external SIEM tools, because default ACA logging may not cover Dynamic Session content or model inference metadata.
