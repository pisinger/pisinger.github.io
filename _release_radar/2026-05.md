## ⛵ AKS

| Indicator | Feature | Type | Description |
|---|---|---|---|
| 🔴 | CVE-2026-31431 (Copy Fail) | Security | Local privilege escalation vulnerability in the Linux kernel `algif_aead` module (CVSS 7.8) allows pod-to-root escalation on affected nodes. Mitigation is globally deployed in node image versions 202604.13.0 and 202604.24.0; existing nodes require an image upgrade or DaemonSet application. |
| 🔴 | AKS-2026-0004: nodes/proxy RCE Risk | Security | On AKS versions earlier than Kubernetes 1.33, the Container Insights add-on retains `nodes/proxy` GET permissions, creating a known RCE vector via the Kubelet API on port 10250. Upgrade clusters to 1.33+ to mitigate. |
| ⚫ | Ubuntu 22.04 Node Image Deprecation | Deprecation | AKS will end support and security updates for Ubuntu 22.04 on June 30, 2027. Node pool creation and image production will cease; scaling and remediation will fail after April 30, 2028. Migrate to Ubuntu 24.04. |
| ⚫ | Ingress NGINX Retirement | Deprecation | The upstream Ingress NGINX project is retiring. AKS production workloads remain supported through November 2026. New AKS 1.36 Automatic clusters will default to Gateway API; migrate existing NGINX-based application routing to Gateway API. |
| ⚫ | Legacy Grouped Recommendations | Deprecation | Grouped recommendation types in the Azure portal are deprecated and will be removed on July 30, 2026. Individual recommendations are now GA and replace the grouped equivalents. |
| ⚫ | AKS Kubernetes LTS 1.29 | Deprecation | AKS Kubernetes Long Term Support version 1.29 is deprecated; upgrade clusters to a supported version. |
| ⚫ | AKS Kubernetes 1.32 Standard Support | Deprecation | Version 1.32 is now available only through Long Term Support. Transition to an LTS plan or upgrade to a standard-support version. |
| ⚫ | Kubelet Serving Certificate Rotation Tag | Deprecation | The `aks-disable-kubelet-serving-certificate-rotation=true` node pool tag will no longer be supported starting April 1, 2027. KSCR will be enabled by default on new and reimaged node pools. |
| 🟢 | Gateway API-based Application Routing | GA | Gateway API-based ingress for the application routing add-on is now generally available, providing a modern replacement for the retiring NGINX-based ingress. |
| 🟢 | NAT Gateway V2 | Preview | Preview support for AKS-managed NAT Gateway V2 outbound type in supported public Azure regions, with automatic exclusion where unavailable. |
| 🟢 | AKS Automatic System Node Pool Protection | GA | AKS Automatic clusters now block `ClusterRoleBinding` create/update requests targeting configured privileged `ClusterRole`s, reducing privilege escalation via service account impersonation. |
| 🟢 | AKS Managed API Server Guard | GA | A last-resort safeguard for the kube-apiserver under extreme load is now generally available. |
| 🟢 | Disable HTTP Proxy | GA | Disabling HTTP Proxy is now GA and enabled by default for new clusters; proxy variables can be removed from pods and nodes on existing clusters. |
| 🟢 | Premium SSD v2 StorageClass | GA | A preinstalled Premium SSD v2 StorageClass is now available on AKS 1.35 clusters in supported regions. |
| 🔵 | Capacity Based Surge | Update | Node pool upgrades can now use both MaxUnavailable and MaxSurge with capacity-aware fallback for surge values, improving upgrade reliability. |
| 🔵 | kube-proxy Reduced Privileges | Update | `kube-proxy` now uses `NET_ADMIN` and `SYS_RESOURCE` capabilities instead of `privileged: true` on Kubernetes 1.30+. |
| 🔵 | Cilium Updates | Update | Cilium updated to v1.17.10 and v1.18.6 to support Gateway API scenarios and address CVE-2025-64715 and CVE-2026-26963. |
| 🔵 | Istio Revisions | Update | Revisions updated to 1.27.9-2, 1.28.6-1, and 1.29.2-1; `asm-1-29` is now available and `asm-1-26` is deprecated. |
| 🔵 | AKS List Available VM SKUs API | Update | New API to view supported VM SKUs available in a subscription for cluster and node pool creation. |
| 🔵 | Custom kube-reserved/Eviction Config | Update | Preview support for customizing default `kube-reserved` and hard eviction kubelet configuration via the custom node preview feature. |
| 🔵 | AKS-managed GPU Metrics | Update | GPU metrics now supported by default in Azure Managed Prometheus and Grafana dashboards. |
| 🔵 | Artifact Streaming + Pod Sandboxing Validation | Update | Added validation to prevent enabling Artifact Streaming with Pod Sandboxing, which is unsupported. |
| 🔵 | Flatcar Container Linux Retirement | Update | Flatcar Container Linux for AKS (preview) will be retired on June 8, 2026; transition by that date. |
| 🔵 | Teleport Removal | Update | Teleport (preview) has been removed from AKS; migrate to Artifact Streaming or disable the feature header. |

## Security Architect Observations

- **AKS defense-in-depth improvements are significant**: The combination of `nodes/proxy` RCE mitigation (Kubernetes 1.33+ fine-grained Kubelet auth), AKS Automatic managed system node pool protection blocking privileged ClusterRoleBindings, and kube-proxy reduced privileges collectively harden the cluster attack surface. Architects should prioritize cluster upgrades and validate pod security policies against these changes.
- **Ingress NGINX retirement demands migration planning**: With upstream maintenance ending and AKS Automatic defaulting to Gateway API on 1.36+, organizations using application routing must plan ingress controller migrations. Gateway API is now GA and provides a supported path, but configuration differences require testing.
- **Cross-tenant and cross-cloud identity complexity is increasing**: Entra ID's cross-tenant group synchronization, Azure role governance via entitlement management, and Defender for Cloud's multicloud AWS RDS protection all expand the identity trust boundary. Architects should review cross-tenant access policies and ensure least-privilege alignment across clouds.
- **Unified portal strategy is accelerating**: Defender for Cloud's integration into the Defender portal, combined with cloud security reporting and the GitHub Advanced Security runtime-to-code feedback loop, reinforces Microsoft's single-pane-of-glass strategy. Architects should evaluate role-based access and data residency implications of consolidating security operations into the Defender portal.
- **Deprecation velocity requires lifecycle governance**: Multiple AKS deprecations (Ubuntu 22.04, KSCR tags, LTS 1.29, 1.32 standard support, Flatcar, Teleport) and Defender for Cloud grouped recommendations removal create a compressed timeline. Organizations need automated asset inventory and lifecycle tracking to avoid operational surprises.
- **Passkey-first authentication is becoming viable at scale**: Entra ID's expanded passkey policy storage (dedicated 20 KB, 10 profiles), system-preferred first-factor authentication, and registration campaign support indicate Microsoft is clearing blockers for enterprise passkey adoption. Architects should pilot passwordless strategies for high-risk user populations.

## Security Operations Observations

- **AKS CVE-2026-31431 requires immediate node hygiene**: The Copy Fail LPE affects a wide range of node images and allows non-root pods to escalate to root. SOC teams should verify node image versions and track upgrade completion, as existing nodes are not patched in-place.
- **New Defender for Identity Entra ID alerts expand cloud detection coverage**: Alerts for guest-to-member promotion, stolen session cookie replay, randomized user agents, and suspicious third-party MFA method additions improve detection of identity-based attacks. SOC teams should tune these alerts and ensure incident response playbooks cover Entra ID compromise scenarios.
- **Sentinel UEBA and AI playbook generation reduce manual SOAR overhead**: The GA of AI playbook generation and expanded UEBA coverage for Okta V2 and GCP Audit Logs provide more automation surface. SOC teams should review anomaly baselines and test AI-generated playbooks in a non-production environment before broad deployment.
- **Defender XDR automatic device isolation changes incident response playbooks**: With automatic attack disruption now able to isolate devices as an active foothold, SOC teams must define clear escalation criteria and release procedures to avoid business disruption from automated containment actions.
- **Defender for Cloud's Docker Hardened image scanning may increase vulnerability volume**: As scanning coverage expands to hardened images, expect additional findings. SOC and vulnerability management teams should prepare prioritization workflows and communicate with container platform teams before the gradual rollout completes.
- **Purview Data Security Investigations with OCR and custom examinations improve insider threat analysis**: The addition of OCR for image files and custom AI-powered examinations strengthens data loss investigations. SOC and insider threat teams should update evidence handling procedures to incorporate image-based exfiltration detection.
