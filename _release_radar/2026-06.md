## Security Architect Observations

- **Identity detection expansion is the headline this month.** Defender for Identity added 10 new alerts targeting Entra ID elevation abuse, TAP collusion, service principal credential theft, DCSync, and SaaS identity provider attacks (SailPoint). This materially closes detection gaps in two critical identity surfaces — Entra ID and on-prem AD — that were previously underserved. Security architects should update their identity threat models and ensure these alerts are routed into SIEM/SOAR workflows.
- **Defender for Cloud continues to unify multicloud container posture.** The simultaneous extension of VA to runtime-discovered images and node hosts on EKS/GKE, plus Kubernetes misconfiguration enforcement (audit-or-block), delivers near-parity across Azure/AWS/GCP. This enables a single policy-and-remediation plane for container security. Architects should plan to enable agentless scanning on AWS and GCP accounts if not already done.
- **AI agent visibility is becoming a first-class security domain.** Local AI agent discovery and runtime protection on Windows/macOS endpoints (Defender for Endpoint) plus the new `AgentsInfo` advanced hunting table signal a platform-wide shift. As agentic workloads proliferate, architects should define an AI agent governance framework, inventory approved agents, and plan for runtime inspection policies.
- **API security posture management reaches serverless compute.** GA of API security for Function Apps and Logic Apps extends CSPM beyond API Management. Architects should inventory all serverless API endpoints and enable Defender CSPM API security to detect unauthenticated, unencrypted, or dormant APIs.
- **Exposure Management is deepening SaaS asset criticality classification.** The 15 new predefined SaaS classification rules (Entra ID, Purview, Intune, M365 Defender, etc.) help prioritize defense-in-depth investments. Architects should review the classification list against their own critical-asset register and identify gaps.
- **Microsoft Fabric's Real-Time Intelligence and AI data agent investments accelerate.** Fabric Graph (GA), NL2SQL improvements, Code Interpreter for Data Agents, and mTLS for MQTT represent a maturation of the platform. Architects should plan for increased data-surface monitoring requirements as Fabric adoption grows.

## Security Operations Observations

- **Ten new identity alerts require tuning and triage workflow updates.** SOC analysts should familiarize themselves with each alert's logic and expected baseline. Alerts like "Reciprocal Temporary Access Pass creation" and "Suspicious service principal sign-in following credential addition" may have low FP rates but need clear runbooks. The DCSync alert is particularly high-signal for AD environments.
- **Container-level KSPM recommendations will increase alert volume significantly.** The shift from cluster-level to per-container misconfiguration assessments will generate more granular findings. SOC teams should configure scale controls and review the deprecation timeline for cluster-level recommendations to avoid alert fatigue during transition.
- **SQL VA Express Configuration automatic enablement may surface new findings.** Over the next month, subscriptions with Defender for SQL will see automatic enablement. Operations teams should verify that Express Configuration does not conflict with existing Classic Configuration baselines and review the new unified REST API for automation.
- **Local AI agent runtime protection introduces a new detection surface.** Blocked and audited agent events (prompt injection, unsafe tool calls) will appear as Defender alerts. SOC teams should create dedicated incident correlation rules for AI agent events and train analysts on the agent-loop telemetry schema.
- **New multicloud security recommendations (60+) across AWS and GCP will expand the compliance dashboard.** Teams managing multicloud environments should prioritize the MSK/OpenSearch recommendations (TLS enforcement, public access, audit logging) and the GCP App Engine SSL recommendations as quick wins with high security impact.
- **Purview Endpoint DLP device attributes via Advanced Hunting enables scalable query workflows.** SOC teams investigating DLP incidents can now query device sync status and policy compliance at scale without manual portal exports — integrate into existing advanced hunting workbooks.
- **Salesforce connector enhancements improve OAuth and identity threat detection.** Operations teams managing Salesforce should enable Real-Time Event Monitoring in the Salesforce console to achieve best latency and coverage for OAuth abuse and session hijacking detections.
