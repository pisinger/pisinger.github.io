## Security Architect Observations

- **Defense-in-depth for containers expands significantly**: Kubernetes misconfiguration enforcement (audit/block at admission time), container-level KSPM recommendations, and runtime-discovered image VA on EKS/GKE close gaps in the container security pipeline from admission to runtime across Azure, AWS, and GCP.
- **AI agent attack surface requires new controls**: Local AI agent discovery and runtime protection introduce device-level inspection of agent loops (prompts, tool calls, responses). Architects should plan for agent inventory governance and runtime policy enforcement as AI agent adoption grows.
- **Cross-cloud parity achieved for Kubernetes security**: Node VA, container image VA, and serverless protection now span Azure, AWS, and GCP, enabling unified posture management. The Defender for Open-Source Relational Databases GA on AWS RDS further extends multi-cloud database protection.
- **Confidential Compute GA on ACA enables regulated workloads**: Hardware-based TEE isolation for containerized workloads on Azure Container Apps supports compliance requirements for data-in-use protection in regulated industries.
- **Advanced hunting schema consolidation**: The `AgentsInfo` table replaces `AIAgentsInfo` with a unified schema across all agent types. Queries must be migrated by July 1, 2026, to avoid breakage.

## Security Operations Observations

- **New alert sources from AI agent runtime protection**: Blocked and audited AI agent events appear as alerts in Defender, requiring new incident response playbooks for prompt injection and unsafe agent actions at the device level.
- **Container admission blocking changes SOC workflow**: Kubernetes misconfiguration enforcement in Block mode prevents non-compliant deployments pre-production. SOC teams must coordinate with DevOps to tune rules and handle blocked deployment alerts.
- **Expanded advanced hunting surface**: `CloudAuditEvents`, `CloudDnsEvents`, and `AgentsInfo` tables provide new query paths for cloud infrastructure monitoring and AI agent inventory. SOC analysts should update hunting queries and dashboards.
- **HTTP access logs for ACA enable forensic analysis**: The new `ContainerAppHTTPLogs` diagnostic category provides high-volume request data for incident investigation on Azure Container Apps workloads.
- **Endpoint DLP data now queryable via Advanced Hunting**: The `DlpInfo` column in `DeviceInfo` enables at-scale DLP device configuration and policy sync monitoring without portal exports, improving DLP operational visibility.
- **Sentinel graph tools enable relationship exploration**: The MCP server graph tool collection allows visual exploration of identity-device-threat relationships, supporting faster investigation of complex attack paths.
