# Microsoft Security Release Radar - June 2026

## 🛡️ Microsoft Defender Cloud

| Indicator | Feature | Type | Description |
|---|---|---|---|
| 🟡 | Vulnerability assessment for runtime-discovered container images on EKS and GKE | Preview | Extends vulnerability assessment to runtime-discovered container images on Amazon EKS and Google GKE, providing unified visibility across Azure, AWS, and GCP using the same recommendation model. Requires AWS or GCP onboarded; additional scanning may increase bill. |
| 🟡 | Kubernetes node vulnerability assessment for EKS and GKE | Preview | Extends Kubernetes node (host) OS-level vulnerability assessment to EKS and GKE, bringing parity with AKS. Surfaces an "Upgrade Kubernetes nodes" recommendation when vulnerabilities are detected. Requires agentless scanning. |
| 🟢 | Defender for Open-Source Relational Databases on AWS RDS | GA | Generally available database threat protection and sensitive data discovery for Aurora PostgreSQL, Aurora MySQL, PostgreSQL, MySQL, and MariaDB on AWS RDS. Preview customers automatically transitioned to GA billing starting July 2026; disable the plan to opt out. |
| 🟡 | Container-level misconfiguration recommendations for Kubernetes | Preview | Agentless KSPM recommendations now assess individual containers for issues such as CPU/memory limits, privilege escalation, read-only root filesystem, and automounting API credentials. Cluster-level recommendations will be deprecated at GA and may coexist temporarily during transition. |
| 🟡 | Actionable recommendation to upgrade AKS for system pod vulnerabilities | Preview | New resolvable recommendation replaces the previous non-actionable recommendation for AKS-managed system pod vulnerabilities, providing a clear remediation path and maintaining visibility into cluster exposure. |
| 🟢 | Serverless protection for Azure and AWS | GA | Generally available capability to discover and assess serverless resources across Azure Web Apps, Azure Functions, and AWS Lambda for misconfigurations, vulnerabilities, and insecure dependencies. |

## Security Architect Observations

- **Multi-cloud Kubernetes security expansion:** June 2026 marks a significant push in Defender for Cloud for EKS and GKE parity, with node-level and runtime container vulnerability assessments plus private cluster sensor support. Security architectures managing hybrid Kubernetes estates should align scanning policies and agentless scanner onboarding across all three hyperscalers.
- **Granular KSPM transition:** The shift from cluster-level to container-level KSPM recommendations represents an architectural move toward finer-grained, agentless posture management at the container boundary. Architects should plan policy migrations and prepare for the deprecation of legacy cluster-level controls.
- **AI agent threat surface:** Local AI agent discovery and runtime protection introduce a new endpoint attack surface (coding agents, local runtimes, desktop assistants). Defense-in-depth strategies should now include endpoint controls that inspect agent loops for prompt injection and unsafe tool calls.
- **Cross-cloud serverless coverage:** With serverless protection reaching GA for Azure and AWS, security reference architectures should incorporate unified serverless assessment into CI/CD and runtime guardrails.
- **Billing visibility for preview-to-GA transitions:** The AWS RDS open-source relational database plan automatically began billing at GA. Architect governance models should include checkpoints for preview features nearing GA to avoid unbudgeted cost impacts.

---

## Security Operations Observations

- **New AI agent alert taxonomy:** SOC teams should prepare for a new category of endpoint alerts related to local AI agent runtime protection, including blocked and audited agent loop events. Analyst training on agent-based threat models and prompt injection indicators will be necessary.
- **Increased Kubernetes finding volume:** Expanding vulnerability assessment to EKS/GKE runtime containers and nodes will increase the volume of multi-cloud container findings. Operations teams should tune prioritization workflows and ensure remediation SLAs account for cross-cloud node upgrades.
- **Actionable AKS remediation:** The new resolvable AKS system pod vulnerability recommendation reduces manual investigation by providing a direct upgrade path. SOC and infrastructure teams should integrate this into vulnerability management runbooks.
- **Container recommendation duplication during transition:** Cluster-level and container-level KSPM recommendations will coexist temporarily. Alert fatigue may increase until cluster-level recommendations are retired; tuning suppression rules may be needed.
- **Serverless asset discovery:** Serverless protection GA will surface new assets (Azure Functions, Web Apps, AWS Lambda) and related recommendations. Asset inventory and tagging practices should be reviewed to support accurate scope and ownership.
