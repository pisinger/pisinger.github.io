## Security Architect Observations

- **Defense-in-depth for AI agents**: The simultaneous GA/Preview releases of local AI agent discovery, runtime protection (Windows + macOS), and agent inventory in Defender XDR/Endpoint signal a new attack surface that requires architectural integration into existing endpoint detection and response workflows. Plan for agent loop inspection and blocking policies.
- **Container security expansion**: Defender for Cloud now covers serverless containers (ACA, ACI), runtime-discovered images on EKS/GKE, node VA across all three clouds, and admission-time Kubernetes enforcement. Architect a unified container security policy spanning Azure, AWS, and GCP with consistent audit-to-block gating.
- **Identity detection coverage broadens**: 9 new Defender for Identity alerts (including DCSync, Entra Connect abuse, SailPoint brute-force) and Salesforce connector enhancements in Cloud Apps extend identity threat detection beyond Microsoft-only ecosystems. Review identity provider coverage gaps and enable Real-Time Event Monitoring in Salesforce.
- **Exposure Management critical asset taxonomy**: 15 new predefined SaaS classifications in Defender Exposure Management require Defender for Cloud Apps onboarding. This enables attack path analysis across the Microsoft SaaS estate — prioritize onboarding to close visibility gaps.
- **Multicloud posture management matures**: 60+ new AWS recommendations, GCP networking checks, and expanded cloud scopes (K8s namespace/cluster, multi-cloud registries) provide granular RBAC for security teams. Reassess cloud scope assignments to align with operational boundaries.

---

## Security Operations Observations

- **New identity alert triage**: 9 new Defender for Identity alerts (anomalous activity after Global Admin elevation, reciprocal TAP creation, DCSync, SailPoint brute-force, etc.) require SOC playbook updates and tuning to avoid alert fatigue. Prioritize DCSync and Entra Connect alerts as critical.
- **AI agent monitoring**: Local AI agent discovery and runtime protection generate new alert types in Defender. SOC teams should establish baselines for normal agent behavior, configure block vs. audit modes, and integrate agent alerts into incident correlation workflows.
- **Advanced hunting schema migration**: The `AIAgentsInfo` table deprecation (July 1) requires query updates to `AgentsInfo`. Audit all custom detections, workbooks, and automation using the old table before the cutoff.
- **DLP investigation enrichment**: New Advanced Hunting access to Endpoint DLP device attributes and enhanced Exchange DLP matched conditions in Activity Explorer improve investigation depth. Update DLP incident response procedures to leverage these new data sources.
- **Salesforce threat detection**: The enhanced Salesforce connector with Real-Time Event Monitoring ingestion provides near real-time OAuth and identity threat detection. Enable Real-Time Event Monitoring in Salesforce console and integrate alerts into existing SOAR playbooks.
