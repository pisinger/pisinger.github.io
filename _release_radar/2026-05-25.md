## Security Architect Observations

| Dimension | Observation |
|---|---|
| Container | Major container security wave: antimalware GA, DNS detection GA, Bottlerocket support for runtime protection, Docker Hardened scanning, and the AKS CVE-2026-31431 requiring urgent node image patching. All three major Kubernetes platforms (AKS, EKS, GKE) now have equivalent runtime protection. Helm-based sensor deployment is now the standard. |
| Platform Unification | Defender for Cloud's integration into the Defender portal reaches GA, unifying cloud security posture management, risk-based secure score, and threat protection in a single console. Combined with cloud security reporting, this consolidates CNAPP workflows into the Defender portal. |
| Identity | Defender for Identity continues to expand its detection surface with seven new Entra ID-specific security alerts covering session cookie theft, Conditional Access bypass, and MFA abuse. Sensor v3.x now supports all identity roles (Entra Connect, AD FS, AD CS) and sensor capacity has tripled. Identity Explorer tab and account correlation rules (Preview) from April expand investigation capabilities. |
| Breaking Changes | Unusually high breaking-change density this window: AKS has a critical CVE, Ingress NGINX retirement, two K8s version end-of-life events, and a new Gateway API prerequisite for mesh membership. Defender for Cloud deprecates grouped recommendations. Organizations should prioritize review of these items under Action Items. |
