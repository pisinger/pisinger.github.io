## Security Architect Observations

- **AKS CVE-2026-47774 (Istio)** requires immediate workload pod restart after add-on upgrade — plan for rolling restart of all meshed deployments to trigger sidecar re-injection. The Istio CNI default change in `asm-1-30` also shifts the proxy redirection model for new clusters, which may affect network policy and security boundary design.
- **Defender for Cloud Apps file policy retirement** signals a broader consolidation of data protection into Purview. Architects should map existing file policy governance actions to Purview DLP equivalents and plan the migration timeline, noting the January 2027 deadline.
- **GitHub actions/checkout v7 pwn request protection** is a critical supply-chain security improvement. Organizations pinning to specific SHAs must proactively update; floating major tags will auto-receive the backport. Review all `pull_request_target` workflows for fork-handling patterns.
- **Entra Backup and Recovery GA** provides a built-in safety net for directory objects including Conditional Access policies and authentication policies. Architects should test restore workflows and consider it as a complement to existing disaster recovery procedures for identity infrastructure.
- **Intune advanced capabilities in M365 E3/E5** (Remote Help, Advanced Analytics, Endpoint Privilege Management, Cloud PKI) reduce licensing friction for endpoint security. Architects should evaluate which capabilities to enable and how they integrate with existing endpoint management workflows.

## Security Operations Observations

- **Defender for Identity** adds 10 new security alerts spanning Entra ID (Global Admin elevation, reciprocal TAP, service principal abuse, bulk deletion), Active Directory (DCSync, Entra Connect abuse), and third-party IdPs (SailPoint brute-force). SOC teams should tune alert rules and update playbooks for these new detection surfaces.
- **Defender XDR AI agent runtime protection** on Windows endpoints provides real-time inspection of agent loops with blocking capability. SOC teams should configure alert routing for blocked/audited events and integrate with incident correlation workflows.
- **Intune Multi Admin Approval now applies to Graph API automation calls** — service principals and scripts modifying Intune resources must include approval headers or risk 403 errors. SOC teams should audit automation accounts and update scripts or configure exclusion lists.
- **GitHub Dependabot 3-day package cooldown** reduces supply-chain risk from compromised releases but may delay non-security updates. SOC teams should verify that security update workflows remain unaffected and consider the cooldown window in patch management SLAs.
- **Azure Monitor diagnostic settings billing expansion (June 15)** may increase costs for platform log export. Operations teams should audit diagnostic settings configurations and review which log categories are now billed to avoid budget surprises.
