## Security Architect Observations

- **Agent security is now a first-class concern**: Defender XDR and Defender Endpoint both ship local AI agent discovery and runtime protection (Windows + macOS). Architects should plan for agent inventory management, runtime protection policies, and integration with existing endpoint detection workflows. The new `AgentsInfo` advanced hunting table provides a unified schema — migrate from `AIAgentsInfo` before July 1.
- **Defense-in-depth for container estates expands significantly**: Defender for Cloud adds Kubernetes admission enforcement (audit/block), container-level KSPM misconfiguration recommendations, runtime-discovered image VA on EKS/GKE, and node VA across all three major clouds. This closes a gap between registry-scanning and runtime protection. Plan for potential billing increases as additional image types are scanned.
- **Identity attack surface monitoring reaches new depth**: Defender Identity's 10 new alerts target sophisticated attack patterns — DCSync, Entra Connect abuse, reciprocal TAP creation, and service principal credential addition. Combined with the Salesforce connector enhancements (real-time event monitoring, OAuth governance), identity teams gain detection coverage for previously hard-to-detect lateral paths. The Exposure Management critical asset classifications provide a structured framework for prioritizing identity control plane protection.
- **Serverless and API security posture matures**: Defender for Cloud's GA of API security posture for Functions/Logic Apps, serverless protection for Azure/AWS, and serverless container posture (Preview) means security teams can now assess the full serverless attack surface from a single console. The new ContainerAppHTTPLogs diagnostic category in Azure Container Apps provides the telemetry foundation for API monitoring.
- **Compliance and data protection tooling expands globally**: Purview adds 9 new sensitive information types covering China, Colombia, Russia, Singapore, South Africa, and Ukraine — essential for multinational compliance programs. The new DLP device scoping and external email grounding controls for Copilot address emerging AI data leakage vectors.

---

## Security Operations Observations

- **SOC workflows must incorporate 10 new identity alerts**: The new Defender Identity alerts require tuning and triage playbook updates. Particularly critical: DCSync detection (AD), anomalous activity after Global Admin elevation (Entra ID), and SailPoint brute-force (third-party IdP). These expand coverage across hybrid identity environments and may generate new alert volumes requiring staffing adjustments.
- **AI agent monitoring creates new SOC data streams**: Local AI agent discovery and runtime protection generate alerts for prompt injection and unsafe agent actions. SOC teams need to establish baselines for agent behavior, create new incident response playbooks for AI-specific threats, and integrate `AgentsInfo` advanced hunting queries into existing hunting workflows.
- **Multicloud recommendation volume increases significantly**: 60+ new AWS recommendations plus expanded EKS/GKE coverage will increase Defender for Cloud's recommendation count. SOC teams should prioritize the new actionable "Upgrade AKS for system pod vulnerabilities" recommendation and the container-level KSPM findings, which provide more granular remediation paths than previous cluster-level recommendations.
- **DLP monitoring gains new dimensions**: Purview's Advanced Hunting integration for Endpoint DLP device attributes, device scoping for DLP policies, and enhanced Exchange DLP event matching provide richer data for DLP investigations. SOC analysts can now correlate DLP events with device health and policy compliance status without portal exports.
- **Salesforce threat detection requires configuration action**: The enhanced Salesforce connector requires administrators to enable Real-Time Event Monitoring in the Salesforce console. Without this configuration, the improved OAuth abuse, session hijacking, and credential stuffing detections will not activate. SOC should coordinate with Salesforce admins to enable the required event logging.
- **SQL VA Express Configuration auto-enablement incoming**: Defender for Cloud will automatically enable SQL VA Express Configuration at the subscription level for Azure SQL MI and Synapse workspaces without existing VA configuration. SOC should verify current SQL VA baselines are preserved and review the new unified REST API for consistent management across all SQL resource types.
