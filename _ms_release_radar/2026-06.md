## Security Architect Observations

1. **Defense-in-depth for AI agents accelerates** — Runtime protection (prompt injection blocking at device level), discovery (Windows + macOS agents), and unified schema (`AgentsInfo` table) signal that agent security is becoming a first-class pillar. Architects should plan for agent lifecycle management, inventory, and runtime controls as part of the endpoint security strategy.

2. **Container security posture reaches multicloud parity** — Runtime-discovered image VA on EKS/GKE, node-level VA across all three clouds, and admission-time enforcement with Block mode represent a significant maturation. Architecture teams should standardize on Defender for Containers as the single pane for Kubernetes security across Azure, AWS, and GCP.

3. **SaaS application attack surface is now classified as critical** — The 15 new predefined classifications in Exposure Management (including Entra ID, Exchange Online, Power Platform, Dynamics 365, Purview) provide structured critical-asset definitions. Architects should map these to organizational crown jewels and integrate into attack path analysis workflows.

4. **API security posture expands beyond API Management** — Function Apps and Logic Apps APIs are now discoverable and assessable via Defender CSPM. Teams should inventory unauthenticated, internet-exposed, and dormant serverless APIs and consider adding them to the API security program.

5. **Data security for AI/Copilot reaches GA with new controls** — Purview protections for M365 Copilot Cowork are GA, and DLP can now block external email from Copilot grounding (preview). Compliance architects should evaluate these controls in the context of shadow AI and data governance expansion.

6. **SQL VA Express Configuration auto-enablement shifts management model** — The upcoming subscription-level automatic enablement removes the need for per-resource storage provisioning. Architects should verify that existing Classic Configuration instances are preserved and that the unified REST API fits into automation tooling.

---

## Security Operations Observations

1. **Identity detection library expands significantly** — 11 new Defender for Identity alerts add critical coverage: DCSync, Global Admin elevation anomalies, reciprocal TAP creation, service principal sign-in after credential addition, and SailPoint brute-force. SOC teams should review alert severities, integrate into incident response playbooks, and tune thresholds.

2. **Kubernetes enforcement moves from audit to block** — Misconfiguration enforcement supports admission-time Block mode. SOC workflows should account for potential deployment blocking; establish exception procedures and monitor audit logs to prevent legitimate deployments from being disrupted.

3. **DLP visibility improves with Advanced Hunting and device scoping** — Endpoint DLP device attributes are queryable via Advanced Hunting, and policies can be scoped to dynamic device groups. SOC analysts can now correlate DLP events with device posture data for more precise investigations.

4. **Salesforce OAuth threat detection goes near real-time** — The enhanced connector ingests Real-Time Event Monitoring data within minutes. SOC teams monitoring SaaS identity threats should enable Salesforce Real-Time Event Monitoring and review the new OAuth app governance insights.

5. **Local AI agent discovery creates a new asset class to monitor** — Endpoint-discovered coding agents, IDE extensions, and AI assistants appear in inventory, exposure map, and advanced hunting. SOC teams should establish baselines for expected agents and investigate unknown or unauthorized agents.

6. **Multicloud security recommendations add 60+ new benchmarks** — AWS services across data, IAM, networking, and compute now have Defender CSPM recommendations. SOC teams should assess the initial compliance posture, prioritize critical findings, and plan for ongoing monitoring across AWS and GCP workloads.
