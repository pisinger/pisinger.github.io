# Microsoft Security Release Radar - June 2026

## ⛵ AKS

| Indicator | Feature | Type | Description |
|---|---|---|---|
| 🟢 | Kubernetes version 1.36 GA and LTS | GA | Kubernetes 1.36 is now generally available and supported as a Long Term Support (LTS) version without requiring a preview feature flag. |

| 🔵 | Auto-derived IPv6 pod CIDR for dual-stack Azure CNI | New/Updated | AKS now automatically derives the IPv6 pod CIDR from the pod subnet when creating a dual-stack Azure CNI static block allocation (VnetScale) cluster. |
| 🔵 | Windows gMSA CoreDNS conflict validation | New/Updated | Windows gMSA now validates for CoreDNS conflicts and rejects enabling gMSA when the coredns-custom ConfigMap already defines a server block for the same domain. |
| 🔵 | Reject FIPS enablement on Pod Sandboxing node pools | New/Updated | AKS now rejects enabling FIPS on Pod Sandboxing (Kata) workload runtime node pools at the API with a clear error. |
| 🔵 | Pod Sandboxing support on Standard_DadsV7-series | New/Updated | Pod Sandboxing (Kata) node pools can now be created on Standard_DadsV7-series VM sizes. |
| 🔵 | Migrate AKS Automatic managed system node pool to Base SKU | New/Updated | You can now migrate an AKS Automatic cluster that uses a managed system node pool to the AKS Base SKU. |
| 🔵 | Fixed aks-istio-system namespace Azure Policy exemption | New/Updated | Fixed an issue where the aks-istio-system namespace was not exempted from the Azure Policy add-on when using application routing with Gateway API in Istio mode. |
| 🔵 | Fixed Multiple Standard Load Balancers label validation | New/Updated | Fixed a bug where Multiple Standard Load Balancers rejected valid domain-prefixed label keys in node selectors; validation now follows standard Kubernetes semantics. |
| 🔴 | Windows Server 2022 retirement timeline | Security | Windows Server 2022 retirement extended to June 30, 2028; unsupported after that date and node images removed starting June 30, 2029. Not supported in Kubernetes 1.37+. |
| 🔴 | Azure Service Mesh add-on default behavior changes | Security | The asm-1-30 revision will change the default proxy redirection mechanism to Istio CNI on new installations and adjust gateway pod node preference weights. |
| 🔴 | Istio add-on patch CVE-2026-47774 | Security | Istio revisions asm-1-29 and asm-1-28 upgraded to patches addressing CVE-2026-47774; restart workload pods to trigger re-injection of the updated istio-proxy sidecar. |
| 🔴 | Azure File CSI Driver upgrade | Security | Azure File CSI Driver upgraded to v1.35.4 on AKS 1.35 and 1.36. |
| 🔴 | Azure Blob Storage CSI driver upgrade | Security | Azure Blob Storage CSI driver upgraded to v1.26.14 (K8s 1.33) and v1.27.7 (K8s 1.34+), fixing a regression with blob containers containing `$` in the name. |
| 🔴 | Azure CNI Powered by Cilium update | Security | Cilium, Hubble, and ACNS images updated to v1.17.15 (Kubernetes 1.32) and v1.16.19 (Kubernetes 1.31). |
| 🔴 | Cloud Provider Azure components update | Security | cloud-controller-manager, cloud-node-manager, and health-probe-proxy updated to the June 18, 2026 release across supported Kubernetes versions. |
| 🔴 | AKS Windows node image updates | Security | Windows Server 2022 and 2025 node images updated to the latest builds for June 2026. |
| 🔴 | AKS Azure Linux node image updates | Security | Azure Linux v3.0 node image updated to 202606.08.1. |
| 🔴 | AKS Ubuntu node image updates | Security | Ubuntu 22.04 and 24.04 node images updated to 202606.08.1. |

## Security Architect Observations

- AKS continues hardening the container compute boundary with FIPS 140-3 on Ubuntu 22.04, Trusted Launch, Confidential VMs, and Pod Sandboxing expansions; plan node image and workload isolation accordingly.
- Defender for Cloud's expanded multicloud posture adds 200+ AWS/GCP recommendations and will affect Cloud secure scores as coverage broadens; architect a unified remediation pipeline across Azure, AWS, and GCP.
- Local AI agent discovery and runtime protection on Windows and macOS endpoints introduces a new attack surface; establish agent inventory, governance, and runtime policy baselines before broad adoption.
- The retirement of Defender for Cloud Apps file policies shifts data protection governance to Purview; align DLP architecture and labeling strategies across the tenant to avoid coverage gaps.
- New Kubernetes admission-time enforcement in Defender for Containers enables proactive blocking of non-compliant deployments; design audit-to-block rollouts with DevOps to prevent production pipeline disruption.

## Security Operations Observations

- New advanced hunting tables (DisruptionAndResponseEvents, CloudAuditEvents, CloudDnsEvents, CloudProcessEvents) enrich cross-domain investigations; update hunting playbooks and detection logic to incorporate them.
- Linux MDE improvements to user attribution in sudo/root sessions reduce insider-threat blind spots; verify that elevated-action hunting queries leverage the original login user ID field.
- Identity risk score GA and new Entra ID alerts (anomalous activity after Global Admin elevation, reciprocal TAP creation, etc.) require SOC playbook updates for identity-centric response workflows.
- Salesforce connector real-time event monitoring reduces OAuth abuse detection latency; ensure Salesforce administrators enable Event Manager storage for the relevant events to achieve best coverage.
- Update advanced hunting queries using the AIAgentsInfo table to the new AgentsInfo schema before the July 1, 2026 retirement date to avoid broken detection and reporting queries.
