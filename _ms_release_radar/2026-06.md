## Security Architect Observations

1. **Identity threat detection expansion is significant**: 10 new Defender for Identity alerts (7 for Entra ID, 2 for AD, 1 for SailPoint) represent a major expansion of identity threat coverage. The DCSync detection and Entra Connect account monitoring close critical gaps in hybrid identity security. Architects should review alert enablement and ensure SIEM integration captures these new signal sources.

2. **Container security posture reaches parity across clouds**: Kubernetes node VA and runtime-discovered image scanning now cover EKS and GKE alongside AKS, while container-level KSPM recommendations replace cluster-level findings. This enables consistent multicloud container security policy enforcement. Plan migration from cluster-level to container-level recommendations before cluster-level deprecation at GA.

3. **AI agent security becomes a first-class concern**: Local AI agent discovery and runtime protection on Windows (and now macOS) endpoints, combined with the new AgentsInfo advanced hunting table, require architects to define AI agent governance policies. The runtime protection capability to block prompt injection at the device level should be evaluated for inclusion in endpoint security baselines.

4. **Defender for Cloud expands API and serverless coverage**: API security posture management for Function Apps and Logic Apps, serverless protection GA, and serverless container posture preview significantly broaden the attack surface covered by Defender CSPM. Organizations running serverless workloads should prioritize onboarding these new capabilities.

5. **Purview DLP and eDiscovery enhancements tighten data security**: Device scoping for Endpoint DLP, external email blocking for Copilot grounding data, and Endpoint DLP evidence collection in Data Security Investigations provide new data exfiltration detection and response capabilities. The new eDiscovery HTML conversion for cloud-native formats (.loop, .page) addresses a long-standing gap in review set searchability.

---

## Security Operations Observations

1. **New identity alerts require SOC playbook updates**: The 10 new Defender for Identity alerts — particularly DCSync detection, reciprocal TAP creation, and suspicious service principal sign-ins — require new triage and response playbooks. SOC analysts should be trained on the specific indicators for each alert type, especially the Entra ID-focused detections that differ from traditional AD-based alerts.

2. **Advanced hunting schema migration needed**: The AIAgentsInfo table deprecation (July 1, 2026) in favor of AgentsInfo requires updating all custom queries, dashboards, and automation that reference the old table. Run a search for `AIAgentsInfo` across all hunting queries and detection rules.

3. **Kubernetes admission control changes SOC workflow**: Kubernetes misconfiguration enforcement in Block mode shifts container security from detect-and-respond to prevent-at-deployment. SOC teams should coordinate with platform engineering to define which rules use Block vs. Audit mode and establish exception processes for blocked deployments.

4. **New Defender for Cloud recommendations increase alert volume**: 60+ new multicloud security recommendations, container-level KSPM findings, and serverless container posture checks will increase recommendation volume. Tune severity levels and configure exemption workflows to prevent analyst fatigue while maintaining coverage.

5. **DLP investigation capabilities reduce per-alert triage burden**: Endpoint DLP evidence collection in Data Security Investigations enables aggregate analysis of exfiltration events rather than per-alert triage. SOC teams should establish investigation templates and automation workflows to leverage this capability for bulk DLP incident handling.

6. **Salesforce connector enhancements require configuration**: The enhanced Salesforce connector requires administrators to enable Real-Time Event Monitoring in the Salesforce console for optimal detection coverage. SOC should coordinate with Salesforce admins to enable the necessary event types and validate data ingestion.
