## Security Architect Observations

1. **AKS Istio CVE-2026-47774** requires immediate pod restart for sidecar re-injection — this is a critical supply-chain vulnerability in the service mesh that could enable lateral movement within clusters. Validate that all workload pods across asm-1-28 and asm-1-29 revisions are restarted.

2. **Defender for Identity's 10 new alerts** significantly expand coverage for Entra ID and AD attack paths, including DCSync detection (previously required third-party tools), reciprocal TAP creation (collusion detection), and SailPoint ISC brute-force monitoring. This strengthens defense-in-depth for hybrid identity architectures.

3. **Intune Multi Admin Approval enforcement on automation** changes the security model for programmatic Intune management. Service principals and CI/CD pipelines must now participate in approval workflows, which may break existing automation. Architect exclusion policies carefully to balance security with operational continuity.

4. **Defender for Cloud's Kubernetes misconfiguration enforcement** (audit/block at admission time) introduces a new gating layer for container security. This complements existing post-deployment monitoring but requires careful testing in Audit mode before enabling Block mode to avoid disrupting CI/CD pipelines.

5. **Purview DLP device scoping with dynamic Entra ID groups** enables conditional enforcement based on device platform and user context — a significant improvement for hybrid work environments where policy must differentiate between Windows and macOS endpoints.

6. **Foundry and Logic Apps agentic automation SKUs** introduce new attack surfaces through AI agent orchestration. Ensure that MCP server endpoints, toolboxes, and agent memory stores are governed with proper RBAC, network isolation, and audit logging before production deployment.

## Security Operations Observations

1. **Defender for Identity DCSync alert** fills a critical detection gap for AD credential theft. SOC analysts should prioritize this alert and validate that domain controller audit logging (Directory Service Access) is properly configured.

2. **Defender XDR's `AgentsInfo` table** replaces `AIAgentsInfo` with a unified schema for all agent types. Update all advanced hunting queries referencing `AIAgentsInfo` before July 1, 2026 to avoid broken detections and reporting.

3. **Linux MDE improvements** (user attribution via sudo, IPC hardening against LD_PRELOAD, offline SIU, scheduled scans) significantly enhance Linux endpoint visibility. SOC teams should enable offline SIU and scheduled scans for Linux servers to close monitoring gaps.

4. **Purview Endpoint DLP evidence collection** in Data Security Investigations enables aggregate analysis of exfiltration events rather than per-alert triage. SOC teams should integrate this into investigation workflows for data breach scenarios.

5. **Defender XDR local AI agent runtime protection** on Windows endpoints introduces a new detection surface for prompt injection and unsafe agent actions. Monitor the new alert types and tune false positives before expanding to all endpoints.

6. **Intune Shadow AI detection** (OpenClaw baseline) provides a new mechanism to discover and block unauthorized AI agents on Windows. SOC teams should deploy the properties catalog policy to inventory local AI agents across the fleet.
