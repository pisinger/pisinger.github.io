## Security Architect Observations

- **AKS CVE-2026-47774 requires immediate action**: The Istio service mesh add-on patches for CVE-2026-47774 (asm-1-29.4 / asm-1-28.8) require workload pod restarts to trigger sidecar re-injection. This is a critical operational step that must be coordinated across cluster teams to close the vulnerability window. The AKS component updates also include Azure CNI Cilium, CSI drivers, and Cloud Provider Azure patches — review the full scope of affected clusters.
- **Entra Backup and Recovery is a significant defense-in-depth addition**: Now GA, this built-in capability provides daily automated backups of critical directory objects (users, groups, apps, Conditional Access policies, service principals) with 7-day retention and point-in-time restore. Security architects should validate that restore workflows are tested and that PIM integration for the new SOC Identity Responder role is configured for least-privilege incident response.
- **AI agent security surface is expanding rapidly**: Defender XDR now discovers and provides runtime protection for local AI agents on Windows and macOS endpoints. Intune adds Shadow AI detection/blocking capabilities. Entra ID extends Conditional Access to agent user accounts. Security architects should develop an AI agent governance policy covering discovery, runtime protection, Conditional Access policies, and device compliance requirements for agent-hosting endpoints.
- **Multi-cloud posture management reaches new maturity**: Defender for Cloud's expanded multicloud coverage (200+ new recommendations, ~90 new resource types for AWS/GCP) is now GA and affects Cloud secure score. The new Kubernetes misconfiguration enforcement (audit/block mode) and container-level KSPM recommendations provide proactive guardrails. Architects should reassess multicloud security scoring baselines and plan for the upcoming cluster-level recommendation deprecation.
- **Microsoft Tunnel and VPN architecture gains stricter controls**: Strict Tunnel Mode for Android forces all traffic through the VPN tunnel and blocks traffic when the VPN drops. Combined with the new MTD enhanced permissions model and RHEL 9.7 support, architects should review mobile device VPN architectures and consider enabling Strict Tunnel Mode for high-security cohorts.
- **Purview DLP and eDiscovery enhancements expand investigation capabilities**: Endpoint DLP evidence collection in Data Security Investigations, device scoping for DLP policies, and the new eDiscovery HTML conversion for cloud-native formats (.loop, .page) provide richer investigation workflows. The new DLP condition to block external email from Copilot processing addresses prompt injection risks. Architects should review DLP policy scope and investigation workflows.

## Security Operations Observations

- **New Defender for Identity alerts require tuning and triage workflow updates**: Nine new security alerts across Entra ID (7), Active Directory (2), and SailPoint ISC (1) cover DCSync detection, suspicious Entra Connect authentication, reciprocal TAP abuse, and service principal credential attacks. SOC teams should review alert severity mappings, create detection rules, and ensure coverage for the new NHI inventory (all service principals now visible).
- **Advanced hunting schema changes require query updates**: The `AIAgentsInfo` table is transitioning to `AgentsInfo` with a July 1, 2026 deprecation date. All custom queries, detections, and automation referencing `AIAgentsInfo` must be migrated. Four new GA schema tables (DisruptionAndResponseEvents, CloudAuditEvents, CloudDnsEvents, CloudProcessEvents) provide new detection opportunities for multicloud environments.
- **Sentinel UEBA integration with incident management improves investigation efficiency**: The ability to link behavior results from the `BehaviorInfo` table to incidents in advanced hunting enriches incident context with UEBA data. SOC teams should update playbooks to leverage this integration and train analysts on the new graph tool for visual relationship exploration.
- **MDE Linux visibility improvements enhance insider threat detection**: Better user attribution in Linux security events (file, process, network) now surfaces the original login user ID even through sudo/root elevation. This is exposed in Advanced Hunting and significantly improves the ability to trace elevated actions back to actual user sessions. SOC teams should update Linux monitoring rules to leverage the new user attribution fields.
- **Multi Admin Approval now applies to automation API calls**: MAA enforcement extends to Microsoft Graph API calls from service principals and automation scripts. Operations teams must update automation workflows to include MAA approval headers or configure Exclusions for specific applications. HTTP 403 errors will result from non-compliant calls.
- **Intune Win32 app HTTPS delivery requirement may impact bandwidth**: Managed Win32 app content now requires HTTPS delivery. Organizations using Microsoft Connected Cache without HTTPS configuration will see traffic fall back to the CDN, potentially increasing internet bandwidth usage. Operations teams should verify Connected Cache HTTPS configuration.
