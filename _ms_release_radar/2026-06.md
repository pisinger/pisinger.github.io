## Security Architect Observations

- **AI Agent Visibility Gap**: The rapid expansion of local AI agent discovery and runtime protection across both Windows and macOS signals a new attack surface. Security architects should inventory agent usage, define policies for approved agents, and integrate runtime protection into endpoint hardening standards before prompt injection and unsafe tool call risks proliferate.
- **Multicloud Posture Normalization**: With Defender for Cloud extending vulnerability assessment, node scanning, and misconfiguration enforcement to EKS and GKE alongside Azure, architects should unify Kubernetes security policy design across clouds rather than maintaining siloed approaches per provider.
- **Identity Control Plane Expansion**: The addition of critical SaaS classifications in Exposure Management and new Entra ID / AD alerts in Defender Identity reinforces the need to treat identity infrastructure as Tier-0. Architect compensating controls and segmentation strategies around identity connectors, conditional access, and privileged access workstations.
- **Deprecation Headwinds**: The cluster-level KSPM recommendation deprecation in Defender for Cloud and the AIAgentsInfo table retirement in Defender XDR require timely migration planning. Validate existing dashboards, automation, and compliance reports that depend on deprecated artifacts.
- **Serverless Container Security**: The convergence of ACA sandboxes, serverless container posture, and function-app API security posture management means container security architecture must now explicitly cover event-driven and sandboxed compute models, not just traditional cluster deployments.
- **Confidential Compute for Compliance**: GA of Confidential Compute on Azure Container Apps provides a tangible control for regulated workloads. Architects should map data classification requirements to Confidential Compute deployments and document the isolation boundary for auditors.

---

## Security Operations Observations

- **Salesforce Monitoring Requires Configuration**: The new Salesforce real-time event monitoring detections will not fire unless the Salesforce admin enables Storing data for relevant events. SOC playbooks should include a step to verify this setting during onboarding and track detection latency.
- **Linux Attribution Improvements**: The new user attribution in Linux file, process, and network events closes a visibility gap for sudo/root actions. SOC analysts should update insider threat and lateral movement hunting queries to leverage the original login user ID field.
- **Response Action Precision**: Selective Response Actions for Tier-0 assets means SOCs can no longer assume all response actions apply uniformly. Update runbooks to tag high-value assets and verify which actions are permitted per device group before executing containment.
- **AI Agent Alert Tuning**: Local AI agent runtime protection introduces a new alert category for blocked and audited agent actions. SOC teams should establish baselines for normal agent behavior, tune false positives for legitimate development tools, and correlate agent alerts with identity and network events.
- **Offline Update Workflows**: Offline security intelligence updates for Linux reduce dependency on internet connectivity for signature freshness. Operations teams should validate offline update channels in air-gapped or restricted environments and integrate them into patch management cycles.
- **Graph-Based Investigation**: Sentinel’s new graph tool and Defender XDR’s entity enrichments with threat intelligence both favor graph-style investigations. SOCs should invest in analyst training for graph traversal and update incident response workflows to include TI enrichment checks on IPs, domains, and files.
