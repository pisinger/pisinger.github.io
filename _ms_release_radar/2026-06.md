## Security Architect Observations

1. **Agent identity and access control is a major theme.** Entra ID's new Conditional Access controls for agent user accounts, the SOC Identity Responder role, and Intune's Vulnerability Remediation Agent transitioning to Entra agentic identity all point to a growing need for identity-based guardrails around AI agents. Architects should plan for agent-specific Conditional Access policies, least-privilege roles for SOC identity response, and agentic identity lifecycle management.

2. **Supply chain security hardening accelerates across the stack.** GitHub's npm v12 install-time security defaults (scripts off by default, git/remote dependencies opt-in), the 3-day Dependabot cooldown, safer pull_request_target defaults, and innersource security advisories represent a defense-in-depth approach to software supply chain attacks. Organizations should update CI/CD pipelines, dependency management workflows, and developer training accordingly.

3. **Multicloud security posture reaches parity.** Defender for Cloud's expanded multicloud coverage (200+ new recommendations, 90+ new resource types across AWS/GCP), Kubernetes node VA on EKS/GKE, and runtime-discovered container image VA create a unified multicloud security operations baseline. Architects should reassess Cloud secure score baselines and ensure AWS/GCP connector onboarding is complete.

4. **AI agent discovery and runtime protection becomes a security control plane.** Defender for Endpoint's local AI agent discovery (Windows and macOS) and runtime protection (prompt/tool/response inspection) introduce device-level controls for shadow AI. Combined with Intune's Shadow AI detection for OpenClaw and Purview's DLP for Copilot external email grounding, this creates a multi-layer defense for AI agent usage that spans endpoint, identity, and data protection.

5. **Deprecation and migration timelines require calendar-based planning.** Key deadlines include: Defender for Cloud Apps file policies (Jan 6, 2027 → Purview), AIAgentsInfo table (July 1, 2026 → AgentsInfo), Windows Server 2022 on AKS (June 30, 2028), Apple Intelligence MDM restrictions (→ DDM), and closed Dependabot alert archival (Aug 25, 2026). Architects should create a deprecation tracking calendar and assign migration ownership.

6. **Least-privilege permissions for security tooling are being enforced.** Defender XDR's Phishing/Security Alert Triage Agents now use scoped email permissions, Intune's Multi Admin Approval extends to API automation calls, and the new SOC Identity Responder role provides purpose-built identity response permissions. This trend toward granular, auditable permissions for security operations tooling should inform RBAC design for all security platforms.

## Security Operations Observations

1. **New identity detection coverage requires alert review and tuning.** Defender for Identity added 10 new security alerts spanning Entra ID (Global Admin elevation, reciprocal TAP creation, service principal abuse, bulk deletion) and Active Directory (DCSync, Entra Connect abuse). SOC teams should review these alerts, assess baseline noise levels, and integrate into incident response playbooks.

2. **AI agent monitoring introduces new data sources and workflows.** Defender for Endpoint's local AI agent discovery populates the AI agent inventory, exposure map, and advanced hunting (AgentsInfo table). Runtime protection generates alerts for blocked/audited agent actions. SOC teams should create hunting queries for local AI agent activity, establish alert triage procedures for agent runtime blocks, and monitor the AgentsInfo table transition deadline.

3. **Advanced hunting schema changes require query updates.** The AIAgentsInfo table is being replaced by AgentsInfo (unified schema) with a July 1, 2026 cutoff. SOC teams must audit all custom advanced hunting queries, workbooks, and automation that reference AIAgentsInfo and migrate to the new schema. The four new GA tables (DisruptionAndResponseEvents, CloudAuditEvents, CloudDnsEvents, CloudProcessEvents) expand multicloud hunting coverage.

4. **Endpoint DLP investigation capabilities expand.** Purview's Endpoint DLP evidence collection in Data Security Investigations enables aggregate analysis of exfiltration events instead of per-alert triage. Device attribute data is now queryable via Advanced Hunting. SOC teams should evaluate these new data sources for insider threat investigations and update DLP incident response workflows.

5. **GitHub security alert lifecycle changes impact forensic capabilities.** Starting August 25, 2026, closed Dependabot alerts older than 2 years move to archival storage (CSV download only). SOC teams should archive critical alert data before the cutoff, update any automation relying on REST API access to closed alerts, and plan for the eventual expansion of this policy to other alert types.

6. **Microsoft Entra Backup and Recovery provides new restore capabilities.** The always-on backup of directory objects (users, groups, apps, Conditional Access policies) with 7-day retention enables self-service recovery from accidental changes or malicious updates. SOC teams should test restore workflows, integrate into incident response runbooks for tenant compromise scenarios, and review snapshot difference reporting capabilities.
