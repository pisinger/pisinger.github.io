## Security Architect Observations

- **Defender for Cloud plan deprecation requires migration planning**: Five deprecated plans (AKS, ACR, Key Vault, DNS, ARM) are now blocked via API. Architect migration to Defender for Containers, Defender for Servers, or fixed pricing models. Review all CI/CD pipelines and automation that call the plan-enablement API.
- **AKS Ubuntu 22.04 kernel CVE noise is a Canonical classification issue, not AKS patch lag**: The broad increase in scanner findings stems from Canonical reclassifying old CVEs. Defense-in-depth strategy should include validating against the `linux-azure` package in the Ubuntu CVE Tracker rather than relying on generic scanner mappings. Consider supplementing with runtime threat detection (Defender for Containers) to reduce reliance on pure vulnerability scanning for prioritization.
- **Defender for Identity sensor v3.x migration unlocks automatic RPC auditing**: The automatic RPC auditing capability removes a manual configuration step and improves identity threat detection coverage on domain controllers. Plan the v2.x-to-v3.x migration as a prerequisite for this and future sensor capabilities.
- **Multi-cloud container security reaches parity**: Vulnerability assessment for runtime-discovered images and Kubernetes node assessment now covers EKS and GKE alongside AKS. This simplifies multi-cloud security posture management with a single pane of glass in Defender for Cloud.
- **Purview DLP now extends to AI interactions at the network layer**: Integration with Entra Global Secure Access enables network-level inspection of text and AI prompts. Architects should evaluate this for data loss prevention in generative AI usage scenarios, particularly for sensitive data shared through browser-based AI platforms.

## Security Operations Observations

- **Canonical CVE reclassifications will inflate vulnerability dashboards**: SOC teams should expect a surge in Ubuntu 22.04 kernel CVE counts across vulnerability scanners. Tune dashboards to distinguish between AKS-specific (`linux-azure`) and generic (`linux`) package mappings. Prioritize validated, exploitable CVEs over classification-noise findings.
- **Defender for Identity sensor migration enables richer domain investigation**: The GA of the Domain investigation page and automatic RPC auditing gives SOC analysts deeper visibility into Active Directory trust relationships, service accounts, and group policies without manual sensor configuration.
- **Prompt injection detection adds a new email threat vector to monitor**: Defender for Office 365's new prompt injection protection requires SOC teams to review and tune detection rules for AI-targeted email attacks. Review alert severity and response playbooks for this new detection type.
- **GitHub secret scanning webhook now distinguishes secret categories**: The new `secret_category` field in the webhook payload allows SOAR workflows to triage default vs. generic/AI-detected secrets differently, reducing false-positive handling for generic detections.
- **Exposure Management's MDASH agentic code scanner introduces a new finding type**: SOC teams should evaluate how code vulnerability findings from MDASH integrate with existing vulnerability management workflows and whether they duplicate or complement existing SAST tooling.
- **Defender Container Sensor security fixes across all versions**: Multiple sensor versions (v0.10.6, v0.9.62, v0.8.55) include patches for credential exposure vulnerabilities. Prioritize sensor updates across all AKS clusters to close these authentication and runtime component vulnerabilities.
