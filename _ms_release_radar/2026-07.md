## Security Architect Observations

- **Defender for Cloud plan consolidation**: The deprecation of five Defender plans (AKS, ACR, Key Vault, DNS, ARM) with API-level enforcement signals a mandatory consolidation toward Defender for Containers, Defender for Servers, and fixed-pricing models. Architect migration paths now to avoid automation breakage; the Defender for Containers plan now covers AKS, ACR, EKS, and GKE, making it the single container security plane.
- **AKS kernel CVE noise from Canonical reclassifications**: The ecosystem-wide reclassification of Ubuntu kernel CVEs creates significant scanner noise that is not indicative of actual AKS patch lag. Security architects should update vulnerability management processes to cross-reference findings against the `linux-azure` package in the Ubuntu CVE Tracker rather than relying on generic `linux` package mappings, and engage Canonical for remediation timelines on unfixed CVEs.
- **Supply chain defense-in-depth deepens**: Multiple GitHub changes (npm v12 install-time security defaults, Dependabot three-day cooldown, actions/checkout pwn request protection, npm preventive account protection) collectively raise the bar against supply chain attacks. Architects should evaluate adopting npm v12's `allowScripts` opt-in model and the Dependabot cooldown across all repositories, and ensure CI/CD pipelines pin actions/checkout to floating major tags to receive security backports automatically.
- **AI security surface expands rapidly**: New AI-specific protections span Defender XDR (Agent 365 security), Defender for Endpoint (AI agent runtime protection), Defender for Office 365 (prompt injection detection), and GitHub Copilot (security reviews, AI code scanning). Architects should inventory AI agent usage across the tenant and plan onboarding to these protections, particularly the Copilot Studio connector for real-time agent protection.
- **Unified RBAC becoming default for MDO**: Defender for Office 365 Plan 2 now defaults to Unified RBAC. Organizations with custom RBAC models should validate compatibility and plan migration before the default becomes enforced for existing tenants.
- **OT/IT convergence in exposure management**: New OT data connectors (Armis, Dragos, Forescout) in Security Exposure Management enable unified vulnerability management across IT and OT environments. Architects should evaluate integrating existing OT asset management tools to close visibility gaps.

## Security Operations Observations

- **AKS CVE triage process change**: The Canonical reclassification flood means SOC teams cannot rely on scanner-reported CVEs against the generic `linux` package for AKS Ubuntu 22.04 nodes. Update triage playbooks to check the installed `linux-azure` package version against the Ubuntu CVE Tracker before escalating; many findings will be unfixed upstream and not actionable.
- **New detection surface for prompt injection**: Defender for Office 365 now detects prompt injection in inbound email. SOC teams should review initial detection fidelity, tune alert rules to reduce false positives, and add prompt injection to phishing investigation playbooks.
- **GitHub secret scanning enrichment**: The new `secret_category` field in webhook payloads and enriched metadata (owner, expiry, project context) enable more precise alert routing and prioritization. Update SOAR playbooks to consume these fields for automated triage and remediation assignment.
- **Insider Risk Management consolidation**: The unified alert experience merges Triage Agent and Standard dashboards. SOC analysts should be trained on the new single-pane view; review any existing custom workflows that depend on the separate dashboards.
- **Dependabot alert archival coming August 25**: Closed Dependabot alerts older than two years will move to archival storage. Export any historical alert data needed for compliance or trend analysis before this date; update any API-based queries that rely on accessing closed alerts beyond the two-year window.
- **npm account takeover safeguards**: The 72-hour read-only state for high-impact npm accounts upon sensitive changes provides a new detection signal. Monitor for alerts from npm about account changes and investigate promptly; the safeguard creates a window for incident response before malicious packages can be published.
