## Security Architect Observations

- **Defender for Cloud plan deprecation requires migration planning**: Five legacy plans (AKS, ACR, Key Vault, DNS, ARM) are now blocked from new onboarding via API. Existing subscriptions continue, but teams must plan migration to Defender for Containers, Defender for Servers, or fixed pricing. Review any automation or IaC templates referencing deprecated plan SKUs.
- **AKS Ubuntu 22.04 kernel CVE noise requires new validation workflow**: Canonical's reclassification of kernel CVEs creates a surge in scanner findings that upgrading alone cannot clear. Security architects should establish a process to cross-reference findings against the exact `linux-azure` package version in the Ubuntu CVE Tracker rather than relying on scanner-reported `linux` package mappings.
- **Container security posture reaches GA across multiple fronts**: Container-level KSPM, Kubernetes admission enforcement, serverless container posture, and multi-cloud (EKS/GKE) vulnerability assessment all reached GA. This significantly strengthens the container defense-in-depth story but requires updating policies and monitoring workflows to cover the new recommendation surface.
- **AI agent and prompt injection attack surface expands**: New protections span Defender XDR (Agent 365), Defender for Endpoint (AI agent runtime), Defender for Office 365 (prompt injection detection), Purview (DLP for AI interactions), and GitHub (AI security detections, prompt injection queries). Architects should map the AI application portfolio and ensure coverage across all layers.
- **OT/IT convergence in Exposure Management**: OT data connectors for Armis, Dragos, and Forescout bring operational technology into the Defender portal. This enables unified vulnerability management but introduces new data ingestion, retention, and compliance considerations for OT environments.

## Security Operations Observations

- **SOC must triage elevated AKS kernel CVE volume**: The Canonical reclassification will generate a high volume of scanner alerts for Ubuntu 22.04 node pools. SOC analysts need a runbook to validate findings against `linux-azure` package versions and distinguish scanner noise from genuine exposure.
- **New Defender for Identity capabilities change DC monitoring**: Automatic RPC auditing on domain controllers (sensor v3.0.8+) and the Domain investigation page provide deeper AD visibility. SOC teams should verify sensor upgrades are complete and review new domain-level telemetry for baseline establishment.
- **GitHub secret scanning enrichment improves alert triage**: Extended metadata (owner, expiry, project context) and the new `secret_category` webhook field enable more precise filtering and prioritization of secret scanning alerts. Update SOAR playbooks to consume these fields.
- **Container admission enforcement shifts left**: Kubernetes misconfiguration enforcement at admission time can block non-compliant deployments. SOC should coordinate with platform teams to define the audit-to-enforce transition window and tune alerting for blocked deployments.
- **Purview Insider Risk Management unified alerting reduces context switching**: The combined alerts list page and expanded user profile details streamline analyst workflows. Review existing IRM alert routing and ensure the new unified experience is incorporated into SOC triage procedures.
