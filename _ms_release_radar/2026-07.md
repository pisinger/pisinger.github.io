## Security Architect Observations

- **Defender plan consolidation accelerates**: The API block on five deprecated Defender plans signals a hard deprecation timeline. Architect migration to Defender for Containers (replacing AKS + ACR plans) and Defender for Servers (replacing DNS plan) should be prioritized. The fixed pricing model for Key Vault and ARM plans requires understanding new cost structures.
- **Container security stack reaches parity across clouds**: With GA of container-level KSPM, Kubernetes misconfiguration enforcement, and vulnerability assessment for EKS/GKE nodes and runtime-discovered images, multi-cloud container security posture is now unified under Defender for Cloud. Attack path analysis for serverless containers (ACA, ACI, ECS Fargate) closes a significant blind spot.
- **AKS kernel CVE noise requires new verification workflow**: The Canonical reclassification wave means vulnerability scanners will report inflated findings against Ubuntu 22.04 nodes. Architects should establish a verification process using the `linux-azure` package version against the Ubuntu CVE Tracker rather than relying on generic scanner mappings. This is an ecosystem-wide issue, not an AKS patch lag.
- **GitHub supply chain defense-in-depth deepens**: npm v12's opt-in script execution, Dependabot's 3-day cooldown, actions/checkout pwn request prevention, and innersource advisories form a layered defense. Architects should evaluate the interaction of these controls with existing CI/CD pipelines and plan for the 2FA-bypass GAT deprecation.
- **OT/IT convergence in exposure management**: The addition of OT data connectors (Armis, Dragos, Forescout) to Security Exposure Management enables unified vulnerability prioritization across IT and OT environments. Architects should assess data ingestion requirements and network segmentation implications.
- **AI security surface expands rapidly**: Defender XDR's Agent 365 protection, Defender for Endpoint's AI agent runtime protection, GitHub Copilot's security review command, and Purview's prompt inspection all address the growing AI attack surface. A cross-team strategy for AI security governance is needed.

## Security Operations Observations

- **SOC workflow for AKS CVE triage must change**: The Canonical reclassification means SOC analysts cannot treat scanner-reported kernel CVEs at face value. Create a runbook to verify `linux-azure` package version against the Ubuntu CVE Tracker before escalating. Consider suppressing generic `linux` package kernel CVEs in favor of `linux-azure`-specific checks.
- **New detection surface for prompt injection in email**: Defender for Office 365's prompt injection protection adds a new alert type. SOC teams should tune alert severity thresholds and integrate with incident response workflows for AI-targeted phishing.
- **Insider Risk Management alert consolidation**: The unified alert experience merges Triage Agent and Standard dashboards. SOC analysts should retrain on the new single-pane view and validate that existing playbooks still trigger correctly.
- **GitHub secret scanning enrichment improves triage**: Extended metadata (owner, expiry, project context) and the `secret_category` webhook field enable faster prioritization. Update SOAR playbooks to consume these new fields for automated enrichment and routing.
- **Dependabot cooldown may delay patch verification**: The 3-day cooldown for version updates (not security updates) means non-critical dependency updates will lag. SOC teams monitoring for dependency freshness should adjust expectations and consider opt-out for high-velocity projects.
- **Defender Identity sensor migration impacts RPC auditing**: Upgrading to sensor v3.0.8+ automatically enables RPC auditing on domain controllers. SOC teams should verify that existing RPC audit configurations are not disrupted and that the increased audit volume is accounted for in SIEM ingestion budgets.
