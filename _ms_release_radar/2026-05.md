## Security Architect Observations

- **AKS defense-in-depth**: The `nodes/proxy` RCE risk (AKS-2026-0004) and CVE-2026-31431 "Copy Fail" LPE together demand immediate AKS node image updates and cluster version upgrades. The new managed system node pools for AKS Automatic add significant security hardening (SSH key blocking, `externalIPs` denial, kube-system secret protection) — evaluate migrating existing Automatic clusters.
- **Entra ID authentication modernization**: SAP SuccessFactors workload identity migration (deadline Nov 2026), TLS 1.0/1.1 deprecation (May 2027), and system-preferred authentication extending to first factor all push toward passwordless, certificate-free, short-lived token architectures. Plan migration roadmaps now.
- **Defender for Cloud multicloud expansion**: Malware detection for EKS/GKE nodes, Docker Hardened image scanning, and Open-Source Relational Databases on AWS RDS GA signal continued investment in multicloud coverage. Review multicloud onboarding and billing implications.
- **Container security maturity**: Defender Container Sensor v0.10 with GA antimalware, Bottlerocket OS support, and ACA integration into Defender for Cloud posture management provide a unified container security surface. Assess coverage gaps across AKS, ACA, EKS, and GKE.
- **Purview DSPM GA**: The new Data Security Posture Management version with administrative unit support and Anthropic Claude connector expands data security governance to AI workloads. Review DSPM configuration for AI application monitoring.
- **Copilot Studio agent governance**: Entra agent identities, agent inventory schema, and consent-based recording provide new governance levers for AI agents. Establish policies for agent lifecycle management and DLP controls.

## Security Operations Observations

- **8 new Defender Identity alerts**: SOC teams must tune detection rules for the new Entra ID alerts, particularly stolen session cookie replay, Conditional Access bypass via non-compliant device, and suspicious third-party MFA method addition — these indicate active post-compromise activity.
- **Automatic attack disruption with device isolation**: The new device isolation capability in Defender XDR automatic attack disruption changes incident response workflows. Review isolation scope, time-limits, and release procedures. Test in non-production first.
- **Defender Chat (Preview)**: SOC analysts gain a natural-language investigation assistant in the Defender portal. Evaluate for Tier-1 triage workflows and ensure data access controls are configured.
- **Advanced hunting Take action on email**: Allow/block top-level domains and attachment hashes directly from query results. Integrate into email threat hunting playbooks for faster containment.
- **Sentinel UEBA GCP anomaly detections**: Five new GCP Audit Logs anomaly detections expand coverage. Review and enable for GCP-connected environments; tune baselines to reduce false positives.
- **Cloud Apps alert suppression**: The new toggle to disable informational alerts for unsanctioned app access helps reduce noise. Review current alert volumes and configure suppression where blocking enforcement is sufficient.
- **Intune MAA enforcement on automation**: HTTP 403 errors may impact existing automation. Audit service principals and scripts for MAA compliance; use the Exclusions tab for critical apps needing immediate remediation.
