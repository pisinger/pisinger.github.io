## Security Architect Observations

1. **AKS nodes/proxy RCE risk (AKS-2026-0004) is the highest-severity architectural concern this month.** The Container Insights add-on's use of `nodes/proxy` GET permission on clusters < 1.33 enables Kubelet API abuse for RCE. The fix requires upgrading to Kubernetes 1.33+ where fine-grained `nodes/pods` subresource (KEP-2862) is available. Defense-in-depth via network restriction on Kubelet port 10250 is recommended during migration.

2. **The convergence of Defender for Cloud into the Defender portal** represents a significant architectural shift for multicloud security operations. CSPM and threat protection are now unified across Azure, AWS, and GCP in a single experience. Security architects should plan for the transition from the Azure portal to the Defender portal as the primary cloud security console, and evaluate the impact on existing SIEM/SOAR integrations and RBAC models.

3. **Multiple AKS OS and add-on deprecations require coordinated migration planning.** Windows Server 2019 (already unsupported), Windows Server Annual Channel (retired May 15, 2026), and Flatcar Container Linux (support ends June 8, 2026) all require migration. The recommended target for Linux is Azure Container Linux (now GA), and for Windows is LTSC. These migrations should be sequenced to avoid compounding operational risk.

4. **The Defender for Cloud + GitHub Advanced Security GA integration** bridges runtime cloud security signals with code-level vulnerability management. Security architects should evaluate the bidirectional sync model for vulnerability prioritization workflows and assess how runtime context (internet exposure, sensitive data access) can drive developer remediation campaigns. Requires DCSPM plan and GHAS license.

5. **TLS 1.0/1.1 deprecation across App Service, Functions, and Logic Apps (May 31, 2027)** gives a 12-month window but requires inventory of all legacy clients and services. The Azure Retirement Workbook provides visibility into affected resources. Architects should prioritize this alongside the Entra Connect Sync enhanced authorization changes that require interactive admin authentication for configuration changes.

## Security Operations Observations

1. **Eight new Defender for Identity alerts targeting Entra ID** significantly expand SOC coverage for cloud identity threats. Alerts for stolen session cookie replay, Conditional Access bypass via non-compliant devices, and suspicious third-party MFA method addition are particularly high-signal. SOC teams should tune these alerts in the Defender portal and review existing playbooks for identity-based incident response.

2. **Automatic attack disruption with device isolation** (Preview in Defender XDR and Defender for Endpoint) introduces automated network containment for compromised devices. SOC teams should review the isolation scope (time-limited, incident-scoped, operator-releasable) and test the feature in a controlled environment before enabling broadly. Consider impact on Tier-0 systems where Selective Response Actions (also new in Preview) may be more appropriate.

3. **The Defender Chat experience** (Preview) provides a natural-language interface for incident investigation. SOC analysts should evaluate its accuracy for common investigation workflows and assess whether it reduces mean-time-to-respond. Be aware that as a preview feature, reliability and coverage may vary.

4. **Advanced hunting Take action wizard now supports allow/block of TLDs and email attachment hashes.** SOC teams can operationalize threat intelligence by blocking malicious TLDs and attachment hashes directly from query results. This reduces the friction between detection and response for email-borne threats.

5. **Sentinel UEBA enhancements** (Okta V2 support, 5 new GCP anomaly detections) expand behavioral analytics coverage. SOC teams should enable the new GCP anomaly detections and validate that the OktaV2_CL table is populated for existing Okta connectors. The consolidated UEBA settings view simplifies configuration management.

6. **Purview DLP changes** — DeepL and Zapier removed from unmanaged AI app support list, and new external domain/user blocking for SharePoint/OneDrive DLP — require SOC and DLP teams to review existing policies. The new per-policy labeling activity monitoring for SharePoint and OneDrive enables proactive identification of labeling failures.
